사용자가 제공한 요구사항을 바탕으로 기능 브랜치를 생성하고 개발 계획을 제시합니다.

요구사항: $ARGUMENTS

다음 순서로 진행하세요:

1. `git branch -a` 로 기존 브랜치 목록을 확인하고 feat/#N 에서 가장 높은 N을 찾아 다음 번호를 결정합니다. 브랜치가 없으면 1부터 시작합니다.

2. 요구사항을 분석해 브랜치 이름을 결정합니다.
   - 형식: `feat/#N-brief-description`
   - brief-description은 영문 소문자 kebab-case, 3단어 이내
   - 예: `feat/#3-add-repo-command`

3. `git checkout -b 브랜치이름` 으로 브랜치를 생성합니다.

4. 요구사항을 구현하기 위한 개발 계획을 체크리스트로 제시합니다.
   - 수정/추가할 파일과 구체적인 변경 내용을 포함합니다.
   - 커밋 단위로 묶을 수 있도록 논리적인 순서로 나열합니다.

5. 커밋 메시지 형식을 안내합니다: `feat/#N-간단한내용`

개발 계획 제시 후 바로 구현을 시작하지 말고 사용자의 확인을 받으세요.
